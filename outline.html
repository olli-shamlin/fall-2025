Course Outline

Session 1: Kick Off

  1. OLLI & course administration items
  2. Common reasons people struggle with devices/apps
  3. Course tenets
  4. Digital life rubric
  5. Review of syllabus & session schedule
  6. Questions are welcome and encouraged!

Session 2: Devices

  1. Device categories
    1. General purpose vs specific/limited purpose
      1. General purpose: desktop/personal computer, laptop, tablet, phone
      2. Specific/limited purpose: kindle, game console, "smart home" devices, etc
    2. Mobile: tablets and phones
    3. "Personal": (also) tablets and phones
    4. Apple, Windows, Chromebook
  2. Anatomy of a device (i.e., internals)
    1. "The Brain": processor(s), memory, storage
    2. Connectivity: Ethernet, Wi-Fi, Bluetooth, cellular, near field communication (NFC)
    3. Interactivity (input/output): screen, speakers, camera, keyboard, mouse, touchpad, touchscreen
    4. Extensibility: ports & peripherals
  3. Compare/contrast common mouse actions to touchscreen gestures

Session 3: Processors, Memory, and Storage

  1. Processors
    1. CPU vs GPU
    2. What are Cores?
    3. Instruction set
    4. How the performance of processors is measured: clock rates & instructions per second
  2. Memory: RAM
  3. Storage: Hard Disk Drivers (HDD) & Solid State Disk Drives (SSDD)
  4. How the size of memory and storage is measured
  5. How the size (power and capacity) of devices is measured

Session 4: Software

  1. Categories: desktop, mobile, web
  2. Software = code + data
  3. Processes: the lifecycle of an app
  4. Concurrency: how devices do more than one thing at a time
    1. Multitasking: multiple apps running concurrently on a single device
    2. Parallel processing: an app doing multiple things at the same time (multi-threading)
    3. Distributed processing: how large computational tasks can be "spread across" multiple devices
  5. More on data
    1. A brief introduction to the binary number system
    2. How characters are represented as binary values
    3. What is "metadata"?

Session 5: Operating Systems

  1. Kinds: macOS, Windows, ChromeOS, iOS, and Android
  2. Primary subsystems: windowing, file system, memory, connectivity (HTTP; TCP/IP)
  3. The relationship between apps and operating systems
  4. How the file system works (understanding documents and folders)
  5. Customizing devices & apps: settings & preferences
  6. User interface: controls (windows, buttons, menus, etc) and icons
  7. Software versioning: the importance of keeping software up to date

Session 6: The Internet

  1. How computers interact with one another: networks and networking
  2. How the internet connects the world: the Internet Backbone
  3. Use cases
    1. Viewing a website page using a Browser
    2. Sending and receiving email
    3. Watching a video
    4. Signing on to a site
    5. Posting, sharing, uploading/downloading
  4. How your home network works
  5. How sites handle multi-user workloads
  6. Cookies: what they are and why sites use/need them

Session 7: The Cloud

  1. What the cloud is (which will also answer "where the cloud is")
  2. Categories
    1. Commercial vs consumer clouds
    2. Primary consumer cloud providers: Apple, Microsoft, and Google
  3. Why use the cloud: key use cases
    1. Storing data in the cloud
    2. Backing up to the cloud
    3. Syncing data across devices via the cloud
    4. Sharing files and collaborating with others
  4. Demonstration of cloud apps

Session 8: Navigating

  1. Browsers
    1. How they work
    2. Types of pages: static, dynamic, & interactive
    3. What your browser remembers: browser history and how to manage it
    4. Cookies: what they are and why they exist
  2. The standard tools for finding things and how they work
    1. Search engines: internet and site
    2. Recommendation engines
    3. Feeds (aka "The Algorithm")
    4. Bookmarks: what they are and how to use them

Session 9: Safety and Privacy

  1. Safety = security + well-being
    1. Encryption
      1. What it is & how it works
      2. Types: strong vs weak
      3. Kinds: at rest, in motion, end-to-end
    2. Relative strengths of the kinds of the kinds of authentication (basic, two-factor, social sign-on, passkeys)
    3. Password managers: when you need one and why
    4. Virtual private networks: when you need one and why
  2. Privacy
    1. How we are identified online (site accounts, fingers, footprints)
    2. Personally Identifying Information (PII)
    3. How sites track your activity
      1. Managing cookies
      2. Tracking blockers
    4. Ad blockers

Session 10: Wrap Up

  1. Review the top level relationships between hardware, software, internet/cloud--maybe this is an "uber" architecture diagram?
  2. Did we achieve the course's learning objectives?
    1. Review course tenets
    2. Review terms and acronyms
  3. Are we better equipped to manage our online experience? (Review the digital life rubric)
  4. Strategies for the next time you are stuck, frustrated, anxious, ...
